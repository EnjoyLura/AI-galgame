# AI制作工业化大型中文Galgame — 完整工作流计划

---

## 一、项目前期规划阶段

### 1.1 游戏概念设计

| 工作项 | 说明 | AI工具方向 |
|--------|------|-----------|
| 主题与类型定位 | 确定游戏题材（校园/奇幻/悬疑等）、目标受众、核心卖点 | LLM头脑风暴、竞品分析 |
| 世界观设定 | 构建完整的世界观框架、历史背景、社会规则 | LLM世界观生成、一致性校验 |
| 核心概念文档 | 编写Game Design Document (GDD) | LLM辅助文档撰写 |

### 1.2 叙事架构设计

| 工作项 | 说明 | AI工具方向 |
|--------|------|-----------|
| 主线剧情大纲 | 设计主线故事弧、起承转合、高潮与结局 | LLM剧情生成、故事结构分析 |
| 分支路线设计 | 规划多路线结构（共通线→个人线→True End）、分支条件、路线依赖关系 | 流程图生成工具、LLM逻辑校验 |
| 选择点设计 | 设计关键选择点、影响因子、路线分支逻辑 | LLM选择后果推演 |
| 剧情数据库 | 建立剧情节点、标签、触发条件的结构化数据 | LLM+结构化存储 |

## 二、角色设计阶段

### 2.1 角色企划

| 工作项 | 说明 | AI工具方向 |
|--------|------|-----------|
| 角色概念设定 | 姓名、年龄、性格、口癖、背景故事、动机 | LLM角色生成、性格一致性维护 |
| 角色关系网 | 设计角色间关系图（恋爱/对立/隐藏关系） | 关系图谱可视化 |
| 路线分支系统 | 定义路线分支条件、选择点逻辑、Flowchart节点 | LLM分支逻辑设计 |
| 角色语音风格 | 每个角色的说话方式、语气词、敬语习惯 | LLM风格化对话生成 |

### 2.2 角色视觉设计

| 工作项 | 说明 | AI工具方向 |
|--------|------|-----------|
| 角色原画设定 | 人设图、三视图、配色方案 | Stable Diffusion / ComfyUI / Midjourney |
| 表情差分设计 | 每角色8-15种表情（喜/怒/哀/羞/惊等） | AI绘图+ControlNet表情控制 |
| 姿势差分设计 | 站立/坐姿/特殊姿势 | AI绘图+OpenPose控制 |
| 服装差分设计 | 常服/制服/私服/特殊服装 | AI绘图+IP-Adapter角色一致性 |
| 角色立绘整合 | 将差分整合为游戏可用格式（PNG分层） | 图像处理脚本 |

## 三、美术资源生产阶段

### 3.1 背景美术

| 工作项 | 说明 | AI工具方向 |
|--------|------|-----------|
| 场景清单 | 根据剧本列出所有需要的背景场景（30-80张） | LLM从剧本自动提取场景 |
| 背景图生成 | 校园/街道/室内/自然等场景 | AI绘图+风格一致性ControlNet |
| 时间差分 | 同一场景的昼/夕/夜版本 | AI绘图+色调控制 |
| 天气差分 | 晴/雨/雪版本 | AI绘图+LoRA风格控制 |

### 3.2 CG插画

| 工作项 | 说明 | AI工具方向 |
|--------|------|-----------|
| CG清单提取 | 从剧本中提取关键剧情CG需求（20-50张） | LLM自动提取关键场景 |
| CG构图设计 | 每张CG的构图、角色位置、氛围 | LLM构图描述+AI绘图 |
| CG精修 | AI生成后的人工精修、细节调整 | Photoshop+AI辅助修图 |

### 3.3 UI/UX设计

| 工作项 | 说明 | AI工具方向 |
|--------|------|-----------|
| 游戏UI设计 | 标题画面、对话框、菜单、存档界面 | AI设计辅助+Figma |
| UI素材切割 | 将设计稿切割为游戏引擎可用素材 | 自动化脚本 |
| 动效设计 | 转场动画、UI交互动效 | LLM生成动画代码 |

## 四、剧本写作阶段（核心）

### 4.1 剧本工业化生产流水线

```
世界观设定 → 路线大纲 → 章节细纲 → 逐场景剧本 → 对话润色 → 一致性校验
```

| 工作项 | 说明 | AI工具方向 |
|--------|------|-----------|
| 共通线剧本 | 共通路线完整剧本（约占40%内容量） | LLM分章节生成+人工审核 |
| 个人线剧本 | 每条角色路线的完整剧本 | LLM风格化写作+角色一致性维护 |
| 支线/隐藏线 | 隐藏剧情、彩蛋内容 | LLM创意生成 |
| 对话润色 | 口语化、角色化、情感化改写 | LLM风格迁移（角色语料微调） |
| 剧情一致性校验 | 检查逻辑漏洞、时间线矛盾、角色OOC | LLM逻辑审查+规则引擎 |
| 文本量规划 | 大型Galgame通常50-150万字 | LLM字数统计与分配 |

### 4.2 剧本格式标准化

| 工作项 | 说明 | AI工具方向 |
|--------|------|-----------|
| 剧本标记语言 | 定义标准剧本格式（角色名/对话/旁白/指令） | 自定义格式规范 |
| 剧本解析器 | 将剧本转换为引擎可读格式 | Python解析脚本 |
| 剧本编辑器 | 可视化剧本编辑工具 | 自研或使用Ren'Py编辑器 |

## 五、音频生产阶段

### 5.1 背景音乐 (BGM)

| 工作项 | 说明 | AI工具方向 |
|--------|------|-----------|
| BGM需求清单 | 根据场景氛围列出所需BGM（20-40首） | LLM从剧本提取氛围需求 |
| BGM生成 | 日常/紧张/悲伤/恋爱/战斗等风格 | Suno / Udio / MusicGen |
| BGM循环处理 | 将AI生成音乐处理为无缝循环 | 音频处理脚本 |
| 音乐风格统一 | 确保整体音乐风格一致 | 统一Prompt模板+后期处理 |

### 5.2 音效 (SE)

| 工作项 | 说明 | AI工具方向 |
|--------|------|-----------|
| 音效清单 | 脚步/开门/雨声/手机等环境音效 | LLM从剧本自动提取 |
| 音效生成/采集 | AI生成或从音效库获取 | AudioCraft / Freesound |
| 音效处理 | 格式转换、音量标准化、淡入淡出 | ffmpeg批处理脚本 |

### 5.3 语音 (CV) — 可选

| 工作项 | 说明 | AI工具方向 |
|--------|------|-----------|
| 角色语音模型训练 | 每个角色的专属TTS模型 | GPT-SoVITS / Bert-VITS2 / Fish-Speech |
| 语音生成 | 从剧本自动批量生成语音 | TTS批量生成脚本 |
| 语音后处理 | 情感调整、语速修正、音量标准化 | 音频处理Pipeline |
| 口型同步 | 语音与立绘口型动画同步 | 自动化口型生成 |

## 六、游戏引擎与开发阶段

### 6.1 引擎选择与搭建

| 方案 | 优点 | 缺点 |
|------|------|------|
| **Ren'Py** | Galgame专用、Python扩展、社区成熟 | 性能有限、3D支持弱 |
| **Unity + Naninovel** | 性能强、跨平台、3D支持 | 学习成本高、授权费用 |
| **Godot + 自研** | 开源免费、轻量灵活 | 需自研VN框架 |
| **自研引擎** | 完全定制 | 开发周期长 |

### 6.2 核心系统开发

| 工作项 | 说明 |
|--------|------|
| 对话系统 | 文字显示、打字机效果、角色名显示、表情切换 |
| 选择分支系统 | 分支跳转、条件判断、变量管理 |
| 路线分支系统 | 选择点跳转、标志位管理、Flowchart可视化、路线锁定/解锁 |
| 存档系统 | 快速存档/读取、存档截图、多存档位 |
| 回想模式 | CG回想、音乐回想、场景回想 |
| 设置系统 | 文字速度、音量控制、分辨率、全屏 |
| 历史记录 | 对话回看、日志系统 |
| 跳过/快进 | 已读跳过、自动播放 |

### 6.3 高级系统（可选）

| 工作项 | 说明 |
|--------|------|
| 动态立绘系统 | 基于表情/姿势参数的立绘切换与过渡动画 |
| 镜头系统 | 画面缩放、平移、震动效果 |
| 粒子特效 | 樱花/雪花/雨滴等氛围粒子 |
| 转场系统 | 多种画面转场效果 |
| 小游戏系统 | 简单互动玩法嵌入 |
| 成就系统 | Steam/平台成就集成 |

## 七、资源管线与自动化阶段

### 7.1 AI资源生产Pipeline

```
Prompt模板库 → AI批量生成 → 自动后处理 → 质量筛选 → 资源入库
```

| 工作项 | 说明 | AI工具方向 |
|--------|------|-----------|
| Prompt工程 | 为每类资源建立标准化Prompt模板 | LLM Prompt优化 |
| 批量生成脚本 | 自动化批量调用AI API生成资源 | Python自动化脚本 |
| 后处理Pipeline | 自动裁剪、缩放、格式转换、命名 | Pillow/ffmpeg脚本 |
| 质量筛选 | AI辅助质量评分+人工审核 | CLIP模型评分 |
| 资源管理数据库 | 统一管理所有游戏资源的元数据 | SQLite/JSON索引 |

### 7.2 一致性保障系统

| 工作项 | 说明 | AI工具方向 |
|--------|------|-----------|
| 角色一致性 | 确保同一角色在不同场景中立绘一致 | IP-Adapter / InstantID / PhotoMaker |
| 风格一致性 | 确保所有美术资源画风统一 | 风格LoRA训练 + 参考图ControlNet |
| 剧情一致性 | 确保剧本无逻辑矛盾 | LLM+知识图谱校验 |
| 世界观一致性 | 设定集规则不被违反 | LLM规则引擎 |

## 十、项目管理维度

| 维度 | 建议 |
|------|------|

## 推荐技术栈总览

```
剧本写作:    LLM (GPT-4/Claude) + 自定义Prompt Chain
角色立绘:    Stable Diffusion + ControlNet + IP-Adapter + LoRA
背景美术:    Stable Diffusion + 风格LoRA + ControlNet
CG插画:      SD + ComfyUI工作流 + 人工精修
UI设计:      Figma + AI辅助设计
BGM:         Suno / Udio / MusicGen
音效:        AudioCraft + 音效库
语音:        GPT-SoVITS / Bert-VITS2 / Fish-Speech
游戏引擎:    Ren'Py (推荐) / Unity+NaNinovel / Godot
自动化:      Python脚本 + ComfyUI API + ffmpeg
项目管理:    Git + Notion/飞书 + 资源管理数据库
```

---

> **核心原则：** 每个阶段都可以通过AI工具大幅提效，但关键决策点（剧情质量、角色魅力、情感体验）仍需人工把控。
