# 《君と描くパレット》— 选择点设计

## 1. 选择系统总览

### 1.1 设计理念

本作不设好感度数值系统，所有路线分支通过**固定选项**直接决定。玩家的每一个选择都是"你想怎么回应"，而非"哪个分数高"。选择的影响是即时的——你选了和谁在一起，就走向谁的故事。

### 1.2 选择点分类

| 类型 | 说明 | 影响 | 出现频率 |
|------|------|------|----------|
| 日常选择 | 放学后去哪里、和谁互动 | 决定后续可用的剧情路径 | 每日1次 |
| 对话选择 | 对话中的回应方式 | 决定当前场景的走向和后续分支 | 每场景0-2次 |
| 关键选择 | 剧情分歧点 | 直接决定路线走向或结局 | 每章1-2次 |

### 1.3 Flowchart可视化

游戏内置flowchart系统，以流程图形式展示全部剧情节点和分支：

- 已通过的节点高亮显示
- 未解锁的节点显示为锁定状态
- 每个选择点标注选项和对应走向
- 玩家可从flowchart直接跳转到已解锁的任意节点
- True End需要5条个人线Good End全部通关后解锁

## 5. Flowchart节点设计

### 5.1 节点状态

| 状态 | 显示 | 说明 |
|------|------|------|
| 已解锁 | 高亮 | 玩家已通过的剧情节点 |
| 已解锁（当前） | 特殊高亮+脉冲 | 当前正在游玩的节点 |
| 可解锁 | 暗色 | 前置条件已满足，尚未游玩 |
| 锁定 | 灰色+锁图标 | 前置条件未满足 |
| 隐藏 | 不可见 | 需要特殊条件才显示在flowchart上 |

### 5.2 节点信息

每个flowchart节点包含以下信息：

| 信息 | 说明 |
|------|------|
| 节点ID | 唯一标识（如C01_03） |
| 场景名 | 简短描述 |
| 章节归属 | 所属章节 |
| 选择点 | 该节点的选项和走向 |
| 已解锁CG | 该节点关联的CG缩略图 |
| 前置节点 | 连线来源 |

### 5.3 Flowchart交互

- 点击已解锁节点可跳转到该节点重玩
- 点击锁定节点显示解锁条件提示
- 支持按章节/路线筛选显示
- 支持缩放和拖拽浏览全局flowchart
- 标注当前路线的推荐游玩路径

## 6. 选择点设计原则

| 原则 | 说明 |
|------|------|
| 无数值累积 | 选择直接决定走向，不累积好感度或依赖度 |
| 情感驱动 | 选择应基于"你想怎么回应"而非"哪个是正确答案" |
| 后果延迟 | 选择的影响不立即显现，而是在后续剧情中逐步体现 |
| 选项即叙事 | 每个选项本身就是叙事的一部分，反映悠真的态度 |
| 角色一致性 | 选择应符合角色的性格和当前关系阶段 |
| 避免惩罚感 | Bad End不是惩罚，而是"不够勇敢"的自然结果 |
| Flowchart引导 | 玩家可通过flowchart回溯和规划，降低重复游玩的挫败感 |
